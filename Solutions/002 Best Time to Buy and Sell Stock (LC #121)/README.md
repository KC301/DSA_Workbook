> You are given an array prices where prices[i] is the price of a given stock on the ith day.

You want to maximize your profit by choosing a single day to buy one stock and choosing a different day in the future to sell that stock.

Return the maximum profit you can achieve from this transaction. If you cannot achieve any profit, return 0.



```

Output: 5

Example 2:

Input: prices = [7,6,4,3,1]
Output: 0
Explanation: In this case, no transactions are done and the max profit = 0.
 

Constraints:

1 <= prices.length <= 105
0 <= prices[i] <= 104
```
 
> You may assume that each input would have exactly one solution, and you may not use the same element twice.

---

### üß± Level 1: Brute Force üê¢

**Approach:**  
Try every pair of buy and sell days and compute the profit.
### C++ (Level #1)
```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int maxProfit = 0;
        int n = prices.size();
        for (int i = 0; i < n - 1; ++i) {
            for (int j = i + 1; j < n; ++j) {
                int profit = prices[j] - prices[i];
                maxProfit = max(maxProfit, profit);
            }
        }
        return maxProfit;
    }
};
```

‚è± Time: `O(n¬≤)`  
üß† Space: `O(1)`  
‚úÖ Works for all LeetCode cases, but not efficient.

---

### ‚öôÔ∏è Level 2: One-Pass Optimized üí° *(Best Practical)*

**Approach:**  
Track the minimum price as you iterate, and compute profit if selling on the current day.
### C++ (Level #2)
```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int minPrice = INT_MAX;
        int maxProfit = 0;

        for (int price : prices) {
            minPrice = min(minPrice, price);  // buy low
            maxProfit = max(maxProfit, price - minPrice);  // sell high
        }

        return maxProfit;
    }
};
```

‚è± Time: `O(n)`  
üß† Space: `O(1)`  
üî• Optimal, clean, readable.

### üöÄ Level 3: Kadane‚Äôs Algorithm Variant ‚ö°

**Approach:**

### üîç Approach:

    max_profit(Prices, infinity, 0).

    MaxProfit;

    max_profit(Rest, P, MaxProfit);

## üöÄ Level 3: Kadane‚Äôs Algorithm Style ‚ö°

### üîç Approach:
Apply maximum subarray sum idea to daily deltas.
### Eerlang (Level #3)
```erlang
-module(stock121).
-export([max_profit_kadane/1]).

max_profit_kadane(Prices) ->
    Diffs = diffs(Prices),
    kadane(Diffs, 0, 0).

diffs([_]) -> [];
diffs([A, B | Rest]) ->
    [B - A | diffs([B | Rest])].

kadane([], _, MaxSoFar) ->
    MaxSoFar;
kadane([X | Rest], MaxCurr, MaxSoFar) ->
    NewCurr = max(0, MaxCurr + X),
    NewMax = max(MaxSoFar, NewCurr),
    kadane(Rest, NewCurr, NewMax).
```

‚è± Time: `O(n)`  
üß† Space: `O(1)`  
üíé Functional Kadane‚Äôs ‚Äî perfect for Erlang‚Äôs recursion.

  def max_profit_brute(prices) do
    for i <- 0..(length(prices) - 2),
        j <- (i + 1)..(length(prices) - 1),
        into: [],
        do: Enum.at(prices, j) - Enum.at(prices, i)

  end
end

## ‚öôÔ∏è Level 2: One-Pass Optimized üí°

### üîç Approach:
Track the minimum price seen so far, update max profit on each step.
### Elixir (Level #2)
```elixir
defmodule Stock do
  def max_profit(prices) do
    Enum.reduce(prices, {nil, 0}, fn price, {min_price, max_profit} ->
      cond do
        min_price == nil -> {price, max_profit}
        price < min_price -> {price, max_profit}
        true -> 
          profit = price - min_price
          {min_price, max(max_profit, profit)}
      end
    end)
    |> elem(1)
  end
end
```

‚è± Time: `O(n)`  
üß† Space: `O(1)`  
üî• Clean, idiomatic Elixir.

## üöÄ Level 3: Kadane‚Äôs Algorithm Style ‚ö°

### üîç Approach:
Apply a version of Kadane's algorithm to profit deltas.
### Elixir (Level #3)
```elixir
defmodule Stock do
  def max_profit_kadane(prices) do
    prices
    |> delta_list()
    |> Enum.reduce({0, 0}, fn delta, {current, max_so_far} ->
      new_current = max(0, current + delta)
      {new_current, max(max_so_far, new_current)}
    end)
    |> elem(1)
  end

  defp delta_list(prices) do
    Enum.zip(prices, tl(prices))
    |> Enum.map(fn {a, b} -> b - a end)
  end
end
```

‚è± Time: `O(n)`  
üß† Space: `O(1)`  
üíé Functional dynamic programming.
